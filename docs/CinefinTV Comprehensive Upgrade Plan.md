# CinefinTV Comprehensive Upgrade Plan

**Date:** 2026-04-10
**Target:** Solidifying Beta to Production-Ready Release

This document synthesizes the current known bugs, performance bottlenecks, architecture debt, and UX friction points into a structured, phase-by-step execution plan.

---

## Phase 1: Critical Stability & Blockers (P0)
*These issues actively break core features, updates, or fundamental TV navigation.*

[ ] Fix In-App Updater Pipeline:** Create the missing `updates/version.json` file in the repository root and ensure `publish.sh` keeps it updated. The in-app updater is currently returning HTTP 404s.
[ ] Stabilize Core Data Dependencies: While Compose, Navigation, and tv-material alpha/beta versions are acceptable, audit and upgrade state-critical libraries—specifically datastore-preferences and paging—to stable releases. datastore-preferences is the highest priority to stabilize since it holds the app's encrypted session credentials and playback positions.
[ ] Subtitle Rendering Reliability:** Investigate and fix the bug where subtitles are selected but do not appear. This requires verifying whether Media3 is actually applying the selected track and checking forced/default flags.
[ ] Tab Row Navigation & First-Focus Contracts:** Fix unpredictable focus jumps. Ensure tab row navigation is deterministic, and document a "first-focus" contract for all main screens (Home, Libraries, Details) so the D-pad always lands on the expected element (e.g., the "Play" button).

## Phase 2: Performance & Fluidity (P1)
*Address frame drops and jank on mid-tier Android TV hardware.*

[ ] Player UI Recomposition Splitting:** Refactor `PlayerPlaybackContent` to split UI branches (badges, controls, skip actions) into finer composables with localized state. Throttle progress updates from 250ms to 500-1000ms when player controls are hidden.
[ ] Debounce Home Screen Focus:** Replace the fragile `withFrameNanos` focus hack with a centralized `FocusNavigationCoordinator` to debounce rapid D-pad scroll operations and avoid redundant `scrollToItem` calls.
[ ] Implement Visual Complexity Tiers:** Add a `CardVisualComplexityPolicy` to reduce shadow/glow alpha and replace animated color transitions with instant switches on lower-end devices.
[ ] Global Motion Spec Wiring:** Create a `CinefinMotionSpec` that respects the user's `respectReduceMotion` preference and device performance tier, routing all animation `tween()` durations through it.

## Phase 3: Architecture & Code Health (P2)
*Clean up memory leaks, concurrency risks, and legacy debt.*

[ ] Fix `JellyfinCache` Thread Safety:** Replace the `LinkedHashMap` (which crashes under concurrent `Dispatchers.IO` access) with a `ConcurrentHashMap` or a synchronized block.
[ ] Fix Interceptor Ordering:** Swap the OkHttp interceptor order in `NetworkModule`. Move `NetworkStateInterceptor` *after* `CachePolicyInterceptor` so offline cache serving actually works.
[ ] Isolate Circuit Breaker State:** Move `circuitBreakerStates` in `BaseJellyfinRepository` from the `companion object` to an instance field so different repository subclasses don't cross-contaminate their error states.
[ ] Delete Monolithic `JellyfinRepository`:** Complete the migration by updating `EnhancedPlaybackManager` to only inject `JellyfinStreamRepository` (and any other specialized repos it needs), fully removing the legacy `JellyfinRepository`.
[ ] Log Swallowed Exceptions:** Fix `PlayerViewModel` to log transcoding/playback exceptions instead of silently swallowing them via `runCatching {...}.getOrNull()`.

## Phase 4: UX Polish & Scaffolded Features (P3)
*Wire up features that are partially built and fix presentation glitches.*

[ ] Fix Watch-Progress Stale Data:** Update `CachePolicyInterceptor`'s `isCacheable` check to exclude user-state endpoints (e.g., paths with `/Users/{userId}/Items`). Currently, the blanket 60-second cache prevents "Continue Watching" from updating immediately after an episode finishes.
[ ] Fix Home Section Limits:** Remove or increase the hardcoded `items.take(12)` cap on the Home screen, or add a "Show all" affordance so users don't lose items in large queues.
[ ] Player Subtitle/Audio Popup Redesign:** Tighten row height/spacing, ensure the focused option is visually obvious, and allow a quick back/escape behavior without losing player state.
[ ] Defer Hero Palette Extraction:** Delay theme seed updates on the Home screen until the image decode is complete and the screen is idle, adding a deduplication window to prevent UI churn during rapid scrolling.
[ ] Surface In-App Updates:** Build the actual UI component (dialog or banner) to consume the `UpdateStatus` states generated by the `UpdateManager`.

### UI Polish (P3.5)
*Refine visual consistency, readability, and focus clarity before production launch.*

[ ] Standardize Focus Rings & Glow Language:** Define a single focus style system (stroke width, glow radius, scale bump) and apply it consistently across cards, chips, and buttons so D-pad focus state is always unmistakable.
[ ] Tighten Typography & Spacing Rhythm:** Normalize title/subtitle line heights, clamp multi-line text overflow behavior, and align vertical spacing tokens so rails and detail panels feel visually balanced at 1080p and 4K.
[ ] Improve Skeleton & Loading Surfaces:** Replace abrupt placeholder transitions with low-motion shimmer/fade variants tied to `respectReduceMotion`, and ensure placeholders match final layout dimensions to reduce visual jump.
[ ] Normalize Poster/Card Image Treatments:** Unify corner radii, gradient overlays, and fallback artwork handling so mixed content sources (posters, backdrops, missing art) maintain a cohesive shelf presentation.
[ ] Audit Contrast & Accessibility on TV Panels:** Verify text/icon contrast against dynamic backdrops, enlarge low-legibility metadata chips, and enforce minimum selection/focusable target sizes for remote navigation.

## Phase 5: Testing & Release Hardening (P4)
*Close coverage gaps to prevent regressions.*

[ ] Macrobenchmark & Baseline Profiles:** Add a benchmark module to measure startup, home-scroll, and player-controls interactions. Add Baseline Profile generation to improve cold start and Compose rendering times.
[ ] Business Logic Unit Tests:** Add test coverage for `BaseJellyfinRepository` circuit-breaker recovery, `JellyfinSessionManager` single-flight mutex behavior, and `EnhancedPlaybackManager` direct-play vs. transcode logic.
[ ] Fix Build Tool & Non-UI Versions: Enforce stable versions for build tools and non-UI testing libraries. Specifically, fix the ksp version string in libs.versions.toml so it matches the stable Kotlin convention prefix (e.g., changing 2.3.6 to the proper 2.3.20-1.0.31 format) to avoid silent annotation processing errors. Additionally, move the hardcoded "io.mockk:mockk-android:1.14.9" out of build.gradle.kts into the version catalog.
