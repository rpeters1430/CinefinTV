# CinefinTV: Comprehensive Project Review & Roadmap (March 2026)

## 1. Executive Summary
CinefinTV has evolved into a modern, Compose-based Android TV client for Jellyfin. The recent transition to a unified card strategy (Horizontal for discovery, Vertical for library) and the overhaul of the playback UI have significantly improved the "10-foot" experience. However, several critical UX "rough edges" remain—primarily around navigation focus and metadata density—that prevent it from feeling like a tier-one native app.

## 2. Current Bug Audit (Verified & Remaining)

### High Priority: Navigation & Focus
*   **[BUG] Tab Row Auto-Navigation**: Currently, focusing a tab (scrolling left/right) triggers an immediate screen change. On TV, navigation should only occur on **Click (Center D-pad)** to avoid accidental heavy network loads and disorienting UI jumps.
*   **[BUG] Initial Focus Offset**: Multiple screens (Home, Library, Details) frequently "land" focus on the middle of the page or specific buttons (like "Play") instead of the top-most element. This forces the user to scroll up immediately upon entering a screen.
*   **[BUG] Lateral Focus Leaks**: Scrolling up from a content row sometimes causes the Tab Row to jump to a different tab (e.g., jumping from Home content to the Search tab) instead of just focusing the Home tab.

### Medium Priority: UI & Visuals
*   **[UI] Home Carousel Sizing**: The hero carousel is reported as too tall on certain 720p/1080p displays, pushing the first row of content partially off-screen.
*   **[UI] "Black on Black" Text**: Section titles (e.g., "Recently Added") sometimes use incorrect color tokens, making them invisible against the dark background.
*   **[UI] Left-Side Clipping**: The first item in a horizontal shelf is occasionally clipped by the screen edge when it scales up (1.05x focus scale).

### Low Priority: Content & Metadata
*   **[CONTENT] "5 Collections" Display**: The Home Carousel shows technical item counts (like "5 Collections") which aren't user-friendly. These should be replaced with genre tags or premier dates.
*   **[CONTENT] End-of-Life Logic**: TV shows that have ended should explicitly show "Ended" instead of "2021-Present".

## 4. Architectural Recommendations

### A. Navigation Refactoring
*   **Switch to Navigation Rail?**: While Tabs are classic, a persistent Side Navigation Rail (standard in YouTube/Netflix TV apps) provides more vertical space for content and reduces "scrolling up to navigate" friction.
*   **Strict Click-to-Navigate**: Modify `TabRow` to only call `navController.navigate` inside the `onClick` lambda, never in `onFocus`.

### B. Performance for Large Libraries (2,000+ items)
*   **Paging3 Enforcement**: Ensure all library grids are backed by Paging3. Standard `List` objects of 2,000 items will cause frame drops during D-pad rapid-scrolling.
*   **Coil Optimization**: Implement a custom `DiskCache` and `MemoryCache` specifically tuned for TV (larger image sizes but less frequent changes).

### C. Technical Details & Symbols
*   **Material Symbols**: Integrate `androidx.compose.material:material-icons-extended` more heavily in detail screens to show technical specs (4K, CC, 5.1 Surround) as iconic badges rather than just text.
